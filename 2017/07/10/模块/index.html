先说啥是模块~一大堆文字来袭

模块是源文件和构建设置的集合，允许您将项目分成不同的功能单元。您的项目可以包含一个或多个模块，并且一个模块可以将其他模块用作依赖项。每个模块都可以独立构建、测试和调试。

如果在自己的项目中创建代码库或者希望为不同的设备类型（例如电话和穿戴式设备）创建不同的代码和资源组，但保留相同项目内的所有文件并共享某些代码，那么增加模块数量将非常有用。

Android Studio使用推荐的构建配置创建设备模块，例如使用Android TV模块的Leanback库

是不是感觉看完了还是对模块的概念很迷茫？这是我从文档上copy下来的，理解了好久，觉得这应该在实际写项目中会用上模块，对于项目中的许多耦合性低的功能可以创建不同模块，方便单独管理。

如何创建模块？

  1. 点击 File > New > New Module.
  2. 在Create New Module界面，AS提供了下来设备模块，选择你想要添加的设备。然后点击Next
    • Phone & Tablet Module
    • Android Wear Module
    • Android TV Module
    • Glass Module
  3. 在配置新的模块表单中，输入以下详细信息，点击next
    1. Application Name: 该名称用作新模块的应用启动器图标的标题。
    2. Module Name:此文本用作您的源代码和资源文件可见的文件夹的名称。
    3. Package Name 包名
    4. Minimum SDK ：模块支持的最低Android版本
  4. 剩下的跟创建project类似

Android Studio为新模块创建所有必需的文件，并将新的模块与项目Gradle文件进行同步。为新设备添加模块还会将目标设备的所有依赖关系添加到模块的构建文件中。Gradle项目同步完成后，新模块将显示在左侧的“项目”窗口中。如果没有看到新的模块文件夹，请确保窗口正在显示Android视图（View > Tool Windows > Project）。

添加完成后可以干嘛咩？

添加新模块后，您可以修改模块代码和资源，配置模块构建设置和构建模块。您还可以像任何其他app一样运行和调试模块。
