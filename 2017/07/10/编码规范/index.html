编码规范

Android贡献源码规范

在Android源码网站上看到了贡献源码的时候必须要采用的编码规范，记录下一些自己要注意的，虽然我不可能贡献出源码。。但是既然官方都要求了编码规范，为什么不遵守呢？ 网站链接

  • 使用空格缩进
* 使用**四个空格来缩进块**，而不要使用制表符。

* 使用八个空格来缩进自动换行，包括函数调用和赋值。

  • 遵循字段命名规范

    • 非公开且非静态字段的名称以 m 开头。
    • 静态字段的名称以 s 开头
    • 其他字段以小写字母开头

    • 公开静态 final 字段（常量）为全部大写并用下划线连接 (ALL_CAPS_WITH_UNDERSCORES)

      public class MyClass {
public static final int SOME_CONSTANT = 42;
public int publicField;
private static MyClass sSingleton;
int mPackagePrivate;
private int mPrivate;
protected int mProtected;
}

  • 限制代码行长度

    • 代码中每一行文本的长度都应该不超过 100 个字符
    • 例外情况：
      1. 如果备注行包含长度超过 100 个字符的示例命令或文字网址，那么为了便于剪切和粘贴，该行可以超过 100 个字符。
      2. 导入语句行可以超出此限制，因为用户很少会看到它们（这也简化了工具编写流程）

  • 将首字母缩写词视为字词

    • 在为变量、方法和类命名时，请将首字母缩写词和缩写形式视为字词，使名称更具可读性

Android资源文件命名

这个找不到官方的（可能搜索姿势有问题），就去网上找了些命名方式。
这里看到滴

命名中通常会缩写英文单词，那就看看有哪些常用的英文单词缩写

名称 缩写
icon ic
selector sl
average avg
background bg
buffer bu f
delete del
document doc

包名命名：

包名 内容
activities Activity类
base 基础共享的类
adapter 适配器
tools 公共工具方法类
bean 一组元素类，内有get、set方法
db 数据库操作类
view 自定义view类
broadcast 广播类

不想一一写啦~

小结

命名什么的重要性不言而喻，自己命名不规范的时候写代码写到后面也会很混乱，如果隔一段时间再来看自己的代码很有可能会有打死自己的冲动，我自己的命名感觉就不怎么好，一直以来没去特意遵循什么规范，或者在写代码写急了就随便取个名字，这是不好的习惯，希望自己写完这篇文章之后能够养成良好的编码习惯，下次写代码之前可以先将变量统一按照某个规范来命名。
