理解《程序员修养》其一

发表于 2017-07-10 | 分类于 书籍 | 阅读次数

最近在看GitBook上的程序员的自我修养，在读第一章的字符编码时，参考资料里的字符编码常识及问题解析 这篇博客里有个不是很理解的地方,记录下来，顺便做个读书笔记。

文章大致讲了些字符编码常识和Unicode与UTF-8之间的关系。

常识

  1. 位与字节
    • 位：（bit）是指计算机里存放的二进制值(0/1)
    • 字节：8个二进制位组合成的“位串”称为一个字节
    • 我的补充：1位十六进制等于4位二进制，例如 十六进制 F= 二进制1111。一个字节等于2位十六进制。
  2. 拉丁字符
    • 基础拉丁字母：指常见的”ABCD“等26个英文字母
    • 基础拉丁字符：基础拉丁字母和英语中一些常见的符号（如数字，标点符号）
    • 派生拉丁字母：在欧洲其它一些非英语国家，为满足其语言需要，在基础拉丁字符的基础上，加上一些连字符，变音字符(如’Á’)。

编码标准

  1. 拉丁编码
    • ASCII：美国信息交换标准代码（American Standard Code for Information Interchange）。ASCII编码只支持基础拉丁字符，用一个字节（8个位）来表示一个字符，并保证最高位的取值永远为’0’。
    • ISO 8859：对ASCII字节的第八位进行拓展编码，即其0000000(0x00)-01111111(0x7f)范围段与ASCII保持一致，而10000000（0x80）-11111111(0xFF)范围段被扩展用到不同的字符集，ISO 8859-n与ASCII是兼容的。
  2. 中文编码

  3. Unicode

    • 全世界字符统一的一种编码。批注：那篇博客说Unicode采用4个字节表示一个字符，但是我查阅资料的时候，看到有说Unicode采用2个字节表示一个字符，然后才知道Unicode也是有不同规范的，而UCS-2采用的是两个字节来编码一个字符，Unicode目前普遍采用的是UCS-2。但是汉字太多两个字节表示不了，为了能表示所有汉字，Unicode也有UCS-4规范，UCS-4采用四个字节来编码一个字符

    • 例如：

      微 <->  \u5fae   <->  00000000 00000000 01011111 10101110
  信 <-> \u4fe1   <->  00000000 00000000 01001111 11100001

    • Unicode编码的问题：Unicode对所有的字符编码均需要四个字节，而这对于拉丁字母或汉字来说其前面三个或两个字节均是0, （原文是这样说的，但是就像上面说的那样Unicode不是均采用4个字节的），这对信息存储来说是极大的浪费。另外一个问题就是，如何区分Unicode与其它编码这也是一个问题，比如计算机怎么知道四个字节表示一个Unicode中的字符，还是分别表示四个ASCII的字符呢？

    • 于是就有了UTF-8编码，UTF-8是Unicode的一种实现方式，Unicode的实现方式除了UTF-8还有其它的，比如UTF-16，UTF-32。
      • UTF-8使用一至四个字节为每个字符编码
      • UTF-16使用两个字节编码
      • UTF-32使用四个字节编码
    • UTF-8的编码规则：
      • 规则1：对于单字节字符，字节的第一位为0，后7位为这个符号的Unicode码，所以对于拉丁字母，UTF-8与ASCII码是一致的。
      • 规则2：对于n字节(n>1)的字符，第一个字节前n位都设为1，第n+1位为0，后面字节的前两位一律设为10，剩下没有提及的位，全部为这个符号的Unicode编码。

例如：

’微’的Unicode是’\u5fae’，
二进制表示是”00000000 00000000 01011111 10101110“，
其取值就位于’0000 0800-0000 FFFF’之间，
所以其UTF-8编码为’11100101 10111110 10101110’

我的一个疑惑就在这里，汉字的Unicode编码不是两个字节表示一个汉字吗？为什么UTF-8就是三个字节表示一个汉字呢？

然后终于理解了，虽然UTF-8是Unicode编码的一种实现形式，但是Unicode到UTF-8并不是直接的对应，要通过一些算法和规则来转换。而前面提到的UTF-8的编码规则，对应n个字节的字符，这里不应该是看这个字符在Unicode中是多少个字节，而应该对应上表中十六进制的范围。不能想当然哦~

作为中文使用者需要注意的一点是Unicode(UTF-8)与GBK，GB2312这些汉字编码规则是完全不兼容的，也就是说这两者之间不能通过任何算法来进行转换,如需转换，一般通过GBK查表的方式来进行。

总结

尽信书不如无书，以后看到比较绝对的语句还是要抱有一点怀疑态度然后去进行验证是否正确，不然容易被误导，就像看这篇《字符串常识及问题解析》的时候，里面说Unicode均采用4字节进行编码。。当我在别的地方看到采用2个字节编码的时候顿时懵逼了，这也让自己知道以后写文章的时候还是要严谨一点，不然万一误导到别人了呢。。
