死丢丢jar包重复的大坑

发表于 2017-05-10 | 分类于 Android ERROR | 阅读次数

最近在帮一个有梦想的学弟写一个简单的app，主要是接入高德地图。

以后再找时间写怎么用高德地图进行POI（兴趣点）搜索和路线规划导航。

这篇博客是开给刚刚遇到的一个大坑，因为本宝宝很生气！

不认真听官方文档的话可是会掉坑里的哦

之前看文档上有这样一句话，但是没怎么理会，因为我两个包都需要用到。。。

然后就开始坑了，我同时引入了这两个包，然后运行的时候，死丢丢无情的报错了。因为现在已经解决了，所以上不了图了。

这可怎么办呢？我当时在网上找了好久，没什么解决办法，然后我心一横。包重复是吧，我把重复的东西都删了。。。我这么一想，就大错特错了。。

因为我删了好几十个重复的东西啊！！！

然后还没删完，我一气就把那个jar包某个文件夹全删了（因为重复的文件都在这个文件夹下）

结果报了新的错误。。

Error:Execution failed for task ‘:app:transformClassesWithDexForDebug’.
com.android.build.api.transform.TransformException: com.android.ide.common.process.ProcessException: java.util.concurrent.ExecutionException: com.android.dex.DexException: Multiple dex files define Lcom/autonavi/amap/mapcore/CameraUpdateMessage;

并没有看懂是啥意思。。然后我就放弃删文件这个行为了。。听文档爸爸的话只留一个jar包下来。。一app不留二jar包啊

那现在怎么办？
我看着报错红红的代码，很惆怅，想想要不要厚着脸皮去问某学长。

然后突然灵机一动，我把我需要的文件放到留下的这个jar包里。。

当时需要导包的是这些类。

然后我就把这些文件复制到没被我删掉的jar包里了。

惊奇的发现没报错了，开心，洗澡睡觉啦啦啦。
